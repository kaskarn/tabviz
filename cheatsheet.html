tabviz() core arguments

Argument Default Purpose
data required Source data frame
label NULL Column with row labels
group NULL Column(s) for collapsible row groups
columns required List of col_*() / viz_*() specs
theme web_theme_nejm() Visual theme
interaction web_interaction() Interactivity preset
title, subtitle, caption, footnote NULL Header / footer text
marker_size NULL Constant or column for marker scaling (e.g. "weight") 
tabviz(
  data,
  label = "study",
  group = "category",
  columns = list(
    col_text("category"),
    viz_forest(point = "hr", lower = "lo", upper = "hi",
               scale = "log", null_value = 1),
    col_interval("hr", "lo", "hi", header = "HR (95% CI)"),
    col_pvalue("pvalue", "P")
  ),
  theme = web_theme_nejm()
)

Visualization columns (viz_*)

Focal visualizations with their own axes and scales.

Function Purpose Example
viz_forest() Forest plots with CI viz_forest(point = "hr", lower = "lo", upper = "hi", scale = "log")
viz_bar() Bar charts with axis viz_bar(effect_bar("value"))
viz_boxplot() Box-and-whisker plots viz_boxplot(effect_boxplot(data = "values"))
viz_violin() Violin / density plots viz_violin(effect_violin(data = "scores"))

Forest-axis arguments live inside viz_forest():

Argument Default Purpose
point, lower, upper required Estimate and CI columns
scale "linear" "log" or "linear"
null_value 0 Reference line value
axis_label NULL x-axis title
axis_range data range Two-element numeric
axis_ticks auto Numeric vector of tick positions
axis_gridlines FALSE Draw vertical gridlines
annotations NULL List of refline() / etc.

Data columns (col_*)

Function Purpose Example
col_text() Text values col_text("category")
col_numeric() Formatted numbers col_numeric("n", decimals = 0)
col_interval() Point (CI) format col_interval("hr", "lo", "hi", header = "HR (95% CI)")
col_n() Sample sizes col_n("n", abbreviate = TRUE)
col_percent() Percentages col_percent("rate")
col_events() Events / N format col_events("events", "n")
col_pvalue() P-values with stars col_pvalue("pvalue", stars = TRUE)
col_currency() Money col_currency("price", symbol = "$")
col_date() Dates col_date("enrolled", format = "%b %Y")
col_bar() Inline horizontal bars col_bar("weight")
col_sparkline() Mini charts col_sparkline("trend")
col_heatmap() Color-scale cells col_heatmap("score")
col_progress() Progress bars col_progress("completion")
col_badge() Colored badges col_badge("status")
col_icon() Icons / emoji col_icon("flag")
col_stars() Star ratings col_stars("quality")
col_pictogram() Glyph piles (count or rating) col_pictogram("score", glyph = "person")
col_ring() Donut gauges with thresholds col_ring("temptation", thresholds = 0.5)
col_img() Inline images col_img("logo_url")
col_reference() Truncated links col_reference("doi", show_icon = TRUE)
col_range() Min-max ranges col_range("min", "max")
col_group() Wrap columns under one header col_group("Results", col_interval(...), col_pvalue("pvalue"))

The first positional argument of every col_*() is the data column name (the field). Cell-styling args (bold, color, bg, emphasis/muted/accent/bold/fill, tooltip, formatter, …) accept a column name or a ~ formula.

Row styling arguments

Argument Values Effect
row_type "data", "header", "summary", "spacer" Row behavior
row_italic TRUE / FALSE Italic text (literal)
row_color CSS color Text color (literal)
row_bg CSS color Background color (literal)
row_indent 0, 1, 2, … Indentation level
row_badge, row_icon Text / emoji Badge / icon on the leftmost cell
row_emphasis (semantic) TRUE / FALSE Bold + theme primary
row_muted (semantic) TRUE / FALSE Lighter color, reduced prominence
row_accent (semantic) TRUE / FALSE Bold + theme accent (most common)
row_bold (semantic) TRUE / FALSE Weight bump only, no color change
row_fill (semantic) TRUE / FALSE Bold + strong row-fill background; fg auto-contrasts

All accept column names or formula expressions: row_bold = ~ pvalue < 0.05. The five semantic flags map to bundles on theme$row so they restyle automatically when you swap themes.

Marker styling arguments

Argument Values Effect
marker_color CSS color Fill color
marker_shape "circle" (default), "square", "diamond", "triangle" Shape
marker_opacity 0 to 1 Transparency
marker_size Multiplier or column Size (1 = default; pass a column name to scale by data, e.g. marker_size = "n")

All accept column names or formula expressions: marker_color = ~ ifelse(sig, "green", "gray").

Four journal presets and three illustrative LOTR presets. set_theme(spec, "<name>") switches by string; pass a WebTheme object directly for anything custom.

Theme Style

web_theme_dwarven() Stone neutrals, forge-glow accent, broad serif display

Theme modifiers (fluent)

tabviz(...,
  theme = web_theme_nejm() |>
    set_inputs(
      brand          = "#2563EB",
      accent         = "#1E40AF",
      fonts_body      = "Georgia, serif"
    ) |>
    set_density("comfortable") |>
    set_header_style("bold") |>
    set_spacing(row_height = 28, header_height = 32) |>
    set_theme_field(c("plot", "point_size"), 6) |>
    set_theme_field(c("plot", "line_width"), 1.5) |>
    set_theme_field(c("axis", "gridlines"), TRUE)
)
Modifier What it edits
set_inputs() Tier 1 inputs (primary / secondary / accent + their _deep companions, status_*, series_anchors, slot_style, fonts)
set_density() set_header_style() set_first_column_style() Per-table variant setters (density, header style, first-column style)
set_spacing() Per-token spacing override on top of the density preset
set_theme_field(theme, path, value) Generic deep setter; path walks the theme tree
set_theme() Swap to a different preset (any of the 7 by name) or a WebTheme object

Build a custom theme from scratch

web_theme(
  name = "my-brand",
  inputs = list(
    neutral        = c("#FFF","#FAFAFA","#EEE","#888","#222"),
    brand          = "#1F3A5F",
    accent         = "#B08938",
    fonts_body      = "Charter, Georgia, serif"
  ),
  variants = list(density = "comfortable", header_style = "bold")
)

Fluent post-build modifiers

tabviz(data, label = "study", columns = list(viz_forest(...))) |>
  set_row_style(bold = ~ pvalue < 0.05, emphasis = "is_primary") |>
  set_marker_style(color = ~ ifelse(upper < 1, "#16a34a", "#94a3b8")) |>
  set_column_style("hr", bold = ~ .x < 1, color = ~ ifelse(.x < 1, "green", "red")) |>
  set_theme("nejm") |>
  set_zoom(zoom = 1, auto_fit = TRUE, max_width = 1200, show_controls = TRUE)
Modifier Purpose
set_row_style() Row-level bold / color / emphasis
set_marker_style() Forest marker color / shape / size
set_column_style() Per-cell formatting on a single column
set_theme() Swap the active theme
set_zoom() Zoom controls and auto-fit (replaces old tabviz() zoom args)

Formula expressions (NSE)

Row / marker styling — reference any column:

row_bold = ~ pvalue < 0.05
marker_color = ~ dplyr::case_when(
  upper < 1 ~ "#16a34a",
  lower > 1 ~ "#dc2626",
  TRUE      ~ "#64748b"
)

Cell styling — use .x for the column’s own value:

col_pvalue("pvalue", bold = ~ .x < 0.05)
col_numeric("hr", color = ~ ifelse(.x < 1, "green", "red"))

# Tooltips on hover

  enable_reorder_rows = TRUE,

# Bar chart
viz_bar(

  axis_range = c(0, 100)
)

  effect_boxplot(data = "measurements", color = "#8b5cf6"),
  show_outliers = TRUE
)

# Boxplot from pre-computed stats
viz_boxplot(
  effect_boxplot(min = "min", q1 = "q1", median = "median",
                 q3 = "q3", max = "max")
)

# Violin plot
viz_violin(
  effect_violin(data = "scores", color = "#f59e0b"),
  show_median = TRUE, show_quartiles = TRUE

Annotations

annotations = list(
  refline(0.8, label = "Threshold", style = "dashed", color = "#e11d48")

Split plots

# Single variable split
tabviz(data, label = "study",
       columns = list(viz_forest(...)),
       split_by = "region")

# Hierarchical split with shared axis
tabviz(data, label = "study",
       columns = list(viz_forest(...)),
       split_by = c("region", "country"),
       shared_axis = TRUE)

# Identical column widths across sub-plots (v0.16+)
tabviz(data, label = "study",
       columns = list(viz_forest(...)),
       split_by = "region",
       shared_column_widths = TRUE)

Exporting

p <- tabviz(data, ...)
save_plot(p, "plot.svg")              # Vector
save_plot(p, "plot.png", scale = 2)   # Raster (2x resolution)
save_plot(p, "plot.pdf")              # Print

Pagination

Break long tables across multiple pages. HTML viewer renders one page at a time with prev/next; save_plot(*.pdf) emits a true multi-page PDF.

# Defaults (30 rows / page, group integrity, orphan_min = 3)
tabviz(data, ..., paginate = TRUE)

# Rows-per-page shorthand
tabviz(data, ..., paginate = 50)

# Format presets
tabviz(data, ..., paginate = paginate_letter())
tabviz(data, ..., paginate = paginate_a4(orient = "landscape"))
tabviz(data, ..., paginate = paginate_slide())

# Full control
tabviz(data, ..., paginate = paginate_spec(rows = 25, orphan_min = 5,
                                           break_on = "split"))

# Override per render (keep the spec, change for one save)
save_plot(p, "out.pdf", paginate = paginate_spec(rows = 50))

# Flatten a paginated spec for a PNG without a warning
save_plot(p, "out.png", paginate = NULL)

# Fluent modifier — set / clear pagination on an existing spec
spec |> paginate(rows = 30)
spec |> paginate(NULL)              # turn off

Multi-page PDF merges per-page PDFs via qpdf::pdf_combine() (in Suggests). Missing qpdf falls back to a numbered series with a warning.

Quick patterns

# Log scale for ratios
viz_forest(point = "hr", lower = "lo", upper = "hi",
           scale = "log", null_value = 1)

# Linear scale for differences
viz_forest(point = "md", lower = "lo", upper = "hi",
           scale = "linear", null_value = 0)

# Significant rows in bold
row_bold = ~ pvalue < 0.05

# Color markers by direction
marker_color = ~ ifelse(upper < 1, "#16a34a", "#94a3b8")

# Scale marker size by sample size
tabviz(data, ..., marker_size = "n")

# Column groups
col_group("Treatment", col_n("n_tx"), col_events("ev_tx", "n_tx"))

# Hierarchical grouping
group = c("region", "country")

# Interval column (always explicit)
col_interval("hr", "lo", "hi", header = "HR (95% CI)", separator = " ")

## Pagination

Break long tables across multiple pages. HTML viewer renders one page at a time with prev/next; `save_plot(*.pdf)` emits a true multi-page PDF.

```r
# Defaults (30 rows / page, group integrity, orphan_min = 3)
tabviz(data, ..., paginate = TRUE)

# Rows-per-page shorthand
tabviz(data, ..., paginate = 50)

# Format presets
tabviz(data, ..., paginate = paginate_letter())
tabviz(data, ..., paginate = paginate_a4(orient = "landscape"))
tabviz(data, ..., paginate = paginate_slide())

# Full control
tabviz(data, ..., paginate = paginate_spec(rows = 25, orphan_min = 5,
                                           break_on = "split"))

# Override per render (keep the spec, change for one save)
save_plot(p, "out.pdf", paginate = paginate_spec(rows = 50))

# Flatten a paginated spec for a PNG without a warning
save_plot(p, "out.png", paginate = NULL)

# Fluent modifier — set / clear pagination on an existing spec
spec |> paginate(rows = 30)
spec |> paginate(NULL)              # turn off
```

Multi-page PDF merges per-page PDFs via `qpdf::pdf_combine()` (in Suggests). Missing `qpdf` falls back to a numbered series with a warning.
