Decision register

Status: living ledger (stood up 2026-06-10). The tidy-keeping half of docs/dev/ship-roadmap.md.

The rule: any work that defers a product decision MUST add an entry here instead of deferring silently. Every entry carries a default-if-undecided — what ships if nobody decides — and a decide-by milestone. At each milestone review, entries due are decided (moved to the Decided section with a one-line rationale) or explicitly re-dated. The register must be empty of pre-1.0 rows at ship.

Entry format: ID | question | options | default if undecided | decide by.

Open

D25 Per-token detailed spacing in the Styling tab needs a sanctioned write channel. The settings-redesign layer model puts “detailed spacing (moved from L2)” in Styling, but per-token spacing edits (individual rowHeight / padding / gap tokens) currently go ONLY through writeThemePath/setThemeField — which the DT-11 boundary gate (settings-band-contract.test.ts, marked “still-true” in CLAUDE.md) forbids the panel from calling. Phase 5 shipped the SANCTIONED spacing surface instead: the density preset (Variations) + the continuous density_factor dial (Styling, where the Variations density signpost points) — both theme inputs via setAuthoringInputs, DT-11-clean. The open question is whether per-TOKEN spacing override is worth a new sanctioned verb (a setSpacingOverride(token, value) that re-resolves through inputs, analogous to setThemeRoleOverride) or stays studio/R-only. (a) ship with density+factor only [default] · (b) add a sanctioned per-token spacing-override verb · (c) relax DT-11 for the Styling tab (a) — density + density_factor cover the common case; per-token is expert-grade and R/set_spacing() already serves it post-1.0, on user demand

Process notes

  • The register is for PRODUCT decisions (behavior, scope, defaults), not engineering choices an arc can settle locally.
  • “Default if undecided” is binding: if the decide-by milestone closes without a decision, the default ships. This is what makes deferral safe.
  • Sources feeding the initial population: interactivity-UX arc deferrals, WYSIWYG fidelity pass residuals, 2026-06-10 cleanup audit “kept for decision” items, 1.0 strategy locks.
