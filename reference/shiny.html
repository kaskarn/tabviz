Shiny Integration

  • Show All Code
  • Hide All Code
  • View Source

Functions for embedding tabviz plots in Shiny applications and updating them in place from server code via proxies. The proxy pattern lets every fluent verb dispatch transparently — the same sort_rows() call works on a WebSpec, an htmlwidget, or a live tabviz_proxy.

Output and Render

tabvizOutput()

Create a Shiny output element for a tabviz plot.

tabvizOutput(outputId, width = "100%", height = "400px")
Argument Type Default Description
outputId character(1) required Output variable name.
width character/numeric "100%" Widget width (CSS units).
height character/numeric "400px" Widget height (CSS units).

renderTabviz()

renderTabviz(expr, env = parent.frame(), quoted = FALSE)
Argument Type Default Description
expr expression required Returns a widget from tabviz().
env environment parent.frame() Evaluation environment.
quoted logical(1) FALSE Whether expr is already quoted.

Proxy Updates

tabviz_proxy()

Create a proxy to update a running tabviz widget without re-rendering it. Pass the proxy to any fluent verb (sort_rows(), filter_rows(), add_column(), set_theme(), …) to dispatch an update message to the browser.

tabviz_proxy(id, session = shiny::getDefaultReactiveDomain())
Argument Type Default Description
id character(1) required Widget (output) id.
session Shiny session getDefaultReactiveDomain() Current reactive session.

Returns a tabviz_proxy object. Always create proxies inside a reactive context (observe, observeEvent, …), never at the top of server.

Verbs that accept a proxy

All fluent verbs dispatch on input type: static WebSpec / htmlwidget objects are mutated in R; tabviz_proxy objects send a custom message to the browser. See the full list in Styling and the Fluent API guide.

Category Verbs
Columns add_column(), remove_column(), move_column(), resize_column(), update_column()
Rows sort_rows(), filter_rows(), clear_filters(), select_rows(), move_row()
Data update_data() (proxy requires a WebSpec)
Groups toggle_group()
Cells set_cell(), set_row_label(), clear_edits() (proxy-only)
Global set_theme(), set_zoom(), paginate(), styling set_* verbs
Labels & watermark set_title(), set_subtitle(), set_caption(), set_footnote(), set_watermark()
Painter paint_row(), paint_cell() (apply a semantic token)
Split set_shared_column_widths() (align column widths across sub-plots)

Reading widget state

Tabviz widgets emit per-dimension Shiny inputs as users interact (sort, filter, drag, edit, paginate, …). Two helpers wrap the boilerplate of pulling them all together:

tabviz_state()

Returns a named list of every observability dimension the widget exposes. The envelope (source, ts) is stripped — values only.

tabviz_state(input, id)
Argument Type Description
input Shiny input object Passed through from server
id character(1) Widget output id (the same one passed to tabvizOutput()) 
observe({
  state <- tabviz_state(input, "tbl")
  # state$sort, state$filters, state$collapsed_groups, ...
})

tabviz_state_envelope()

Like tabviz_state() but preserves each dimension’s envelope, so callers see source ("user" / "proxy") and ts alongside the value. Use this to ignore proxy-originated mutations and avoid feedback loops:

observe({
  sort <- input[["tbl_sort"]]
  req(sort, sort$source == "user")
  # react only to user-driven sort changes
})

# Or pull all dimensions at once:
env <- tabviz_state_envelope(input, "tbl")
env$sort$value      # the value
env$sort$source     # "user" or "proxy"
env$sort$ts         # timestamp

Split-Table Bindings

splitTabvizOutput()

splitTabvizOutput(outputId, width = "100%", height = "600px")

Same arguments as tabvizOutput() with a taller default height suited to the split-view’s sidebar layout.

renderSplitTabviz()

renderSplitTabviz(expr, env = parent.frame(), quoted = FALSE)

expr should return a widget created from a split_table() result (or a forest_plot() / tabviz() call with split_by).

split_tabviz_proxy()

split_tabviz_proxy(id, session = shiny::getDefaultReactiveDomain())

Returns a split_tabviz_proxy object.

split_tabviz_select()

Select a sub-plot in a split-table tabviz.

split_tabviz_select(proxy, key)
Argument Type Description
proxy split_tabviz_proxy From split_tabviz_proxy().
key character(1) Sub-plot key (e.g. "Americas" or "Male__Young").

Examples

Basic Shiny app

Code 
library(shiny)
library(tabviz)

ui <- fluidPage(
  titlePanel("Forest Plot Demo"),
  sidebarLayout(
    sidebarPanel(
      selectInput("theme", "Theme:",
        choices = c("Default", "JAMA", "Lancet", "Modern")
      )
    ),
    mainPanel(
      tabvizOutput("plot", height = "500px")
    )
  )
)

server <- function(input, output, session) {
  data <- data.frame(
    study = c("Study A", "Study B", "Study C"),
    hr    = c(0.72, 0.85, 0.91),
    lower = c(0.55, 0.70, 0.75),
    upper = c(0.95, 1.03, 1.10)
  )

  output$plot <- renderTabviz({
    theme <- switch(input$theme,
      "Default" = web_theme_cochrane(),
      "JAMA"    = web_theme_jama(),
      "Lancet"  = web_theme_lancet(),
      "Modern"  = web_theme_cochrane()
    )

    tabviz(data,
      label = "study",
      columns = list(
        viz_forest(point = "hr", lower = "lower", upper = "upper",
                   scale = "log", null_value = 1)
      ),
      theme = theme
    )
  })
}

shinyApp(ui, server)

Proxy updates (no re-render)

Code 
library(shiny)
library(tabviz)

ui <- fluidPage(
  tabvizOutput("plot"),
  actionButton("filter_btn", "Filter HR < 1"),
  actionButton("clear_btn",  "Clear Filter"),
  actionButton("sort_btn",   "Sort HR desc")
)

server <- function(input, output, session) {
  data <- data.frame(
    study = c("Study A", "Study B", "Study C", "Study D"),
    hr    = c(0.72, 0.85, 1.15, 0.91),
    lower = c(0.55, 0.70, 0.90, 0.75),
    upper = c(0.95, 1.03, 1.45, 1.10)
  )

  output$plot <- renderTabviz({
    tabviz(data,
      label = "study",
      columns = list(
        viz_forest(point = "hr", lower = "lower", upper = "upper",
                   scale = "log", null_value = 1)
      )
    )
  })

  proxy <- tabviz_proxy("plot")

  observeEvent(input$filter_btn, filter_rows(proxy, "hr", "lt", 1))
  observeEvent(input$clear_btn,  clear_filters(proxy))
  observeEvent(input$sort_btn,   sort_rows(proxy, "hr", "desc"))
}

Replacing the dataset

update_data() on a proxy requires a WebSpec (not a raw data frame) so the new columns/types can be serialized. Build one with tabviz(..., .spec_only = TRUE).

Code 
observeEvent(input$switch_dataset, {
  new_spec <- tabviz(
    get_dataset(input$dataset_name),
    label = "study",
    columns = list(
      viz_forest(point = "hr", lower = "lower", upper = "upper",
                 scale = "log", null_value = 1)
    ),
    .spec_only = TRUE
  )
  update_data(tabviz_proxy("plot"), new_spec)
})
