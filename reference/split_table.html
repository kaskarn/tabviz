split_table()

  • Show All Code
  • Hide All Code
  • View Source

split_table() slices a spec or widget into a SplitForest — one sub-plot per unique value (or combination of values) of the splitting column(s). The resulting object renders with sidebar navigation in the browser, can be exported one file per sub-plot via save_split_table(), and supports a shared-axis option for cross-subgroup comparison.

The same effect is available inline on tabviz() and forest_plot() via the split_by / shared_axis arguments — split_table() is the explicit constructor used when you want a SplitForest object to inspect, modify, or hand to save_split_table().

Signature

split_table(x, by, shared_axis = FALSE, shared_column_widths = FALSE, ...)

Arguments

Argument Type Default Description
x WebSpec / htmlwidget / data.frame required Source. A data frame is fed through web_spec() first.
by character required Column name(s) to split on. A single string creates one plot per unique value; a vector creates a hierarchical navigation tree (outermost first).
shared_axis logical(1) FALSE When TRUE, all sub-plots use the same x-axis range so subgroup effects can be compared at a glance.
shared_column_widths logical(1) FALSE When TRUE, stamp identical per-column widths across every sub-plot so columns line up tile-to-tile when screenshots are stacked in slides / PowerPoint. Widths are computed once from the combined data. Added in v0.16.
... — — Forwarded to web_spec() when x is a data frame.

Return value

A SplitForest containing one WebSpec per split combination, the navigation tree, the split variables, and (when shared_axis = TRUE) the shared axis range.

tabviz() / forest_plot() shortcut

Both top-level entry points accept split_by and shared_axis directly. They build a WebSpec first, call split_table() internally, then render the result.

# Equivalent
tabviz(data, label = "study",
       columns = list(viz_forest(point = "or", lower = "lo", upper = "hi")),
       split_by = "region", shared_axis = TRUE)

tabviz(data, label = "study",
       columns = list(viz_forest(point = "or", lower = "lo", upper = "hi")),
       .spec_only = TRUE) |>
  split_table(by = "region", shared_axis = TRUE) |>
  forest_plot()

Pagination cascades through splits

Set paginate = ... on the underlying tabviz() call (or pipe paginate() after split_table()) and the spec is propagated into every subview. Each subview paginates independently with its own row-count budget; PDF export emits one combined multi-page PDF per split leaf:

sf <- tabviz(data, label = "study",
             columns = list(viz_forest(point = "or", lower = "lower", upper = "upper")),
             paginate = paginate_letter(),
             .spec_only = TRUE) |>
  split_table(by = "region")

save_plot(sf, "out/region.pdf")  # writes out/Americas.pdf, out/Europe.pdf, ...

Single-subview export with save_plot(which = ...)

save_plot() accepts a SplitForest directly. Without which =, it dispatches to save_split_table() and writes one file per leaf into a directory. With which =, it extracts a single subview by key (e.g. "Male__Young") or 1-based integer index and writes to a single file:

save_plot(sf, "out/europe.svg", which = "Europe")   # named subview
save_plot(sf, "out/first.svg",  which = 1L)         # by index

Examples

Code 
set.seed(42)
data <- data.frame(
  study  = paste0("Study ", 1:30),
  region = sample(c("Americas", "Europe", "Asia"), 30, replace = TRUE),
  or     = exp(rnorm(30, log(0.8), 0.3))
)
data$lower <- data$or * exp(-1.96 * 0.25)
data$upper <- data$or * exp( 1.96 * 0.25)

forest_plot(data,
  point = "or", lower = "lower", upper = "upper",
  label = "study",
  split_by = "region",
  scale = "log", null_value = 1,
  theme = web_theme_nejm()
)
Code 
# Hierarchical split with nested sidebar
forest_plot(data,
  point = "or", lower = "lower", upper = "upper",
  label = "study",
  split_by = c("sex", "age_group"),
  scale = "log", null_value = 1
)
Code 
# Build the SplitForest explicitly to export
sf <- tabviz(data, label = "study",
             columns = list(viz_forest(point = "or", lower = "lower", upper = "upper")),
             .spec_only = TRUE) |>
  split_table(by = c("sex", "age_group"), shared_axis = TRUE)

sf                              # in a knitr chunk, renders as the live widget
save_split_table(sf, "out/")    # one file per sub-plot
NoteSplitForest renders as a widget in knitr chunks

A SplitForest carries a registered knit_print method (since this audit cycle), so a chunk whose last value is sf renders as the interactive split widget rather than dumping the S7 object’s internal structure as raw text. At an interactive prompt outside knitr, print(sf) still prints a one-line summary.

## Signature

```r

```

|---|---|---|---|

| `...` | — | — | Forwarded to `web_spec()` when `x` is a data frame. |
