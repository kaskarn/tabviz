save_plot()

Export a tabviz widget, WebSpec, or SplitForest to a static SVG, PDF, or PNG.

The pipeline runs entirely in R: the spec is serialized to JSON, the bundled JavaScript SVG generator (inst/js/svg-generator.js) is run via V8, and PDF/PNG outputs are produced from that SVG via rsvg.

NoteAspect-ratio first

As of v0.30, save_plot() is aspect-ratio first: the output’s shape is derived from the spec’s natural dimensions, and width / height / ratio are display / aspect levers — not size forcers. Three modes:

  • Natural — no width / height / ratio. Renders at the spec’s natural dimensions.
  • Display-scaled — exactly one of width / height. Same content, just scaled (SVG viewBox keeps coordinates intact). Aspect ratio preserved.
  • Aspect-changed — both width and height, or ratio. Triggers a relayout via the lever ladder. Flex-eligible columns (flex = TRUE, the default for viz_* / forest columns) absorb the width delta; row heights scale to absorb the height delta. Calculated, never SVG-stretched.

save_plot()

save_plot(x, file,
          width = NULL, height = NULL, ratio = NULL,
          flex = TRUE, scale = 2,
          which = NULL, paginate = NULL,
          ...)
Argument Type Default Description
x WebSpec / htmlwidget / SplitForest required The plot to export. SplitForest is dispatched to save_split_table() unless which is set.
file character(1) required Output file path (or directory for SplitForest without which). Extension determines format: .svg, .pdf, or .png.
width numeric(1) / NULL NULL Display width in pixels. NULL preserves natural. With height or ratio, triggers Mode 3 relayout.
height numeric(1) / NULL NULL Display height in pixels. NULL preserves natural. With width or ratio, triggers Mode 3 relayout.
ratio numeric(1) / NULL NULL Target aspect ratio (width / height). When set with one of width / height, the other is computed. When set alone, target width = natural width and target height = natural width / ratio. Falls back to the spec’s target_aspect (set via set_aspect_ratio()) when NULL. Mutually exclusive with passing both width and height.
flex logical / numeric(1) TRUE Lever-ladder cap for flex-column absorption. TRUE = cap of 2 (each flex column may grow / shrink within [0.5×, 2×] of natural). FALSE = disable flex absorption (cap of 1; aspect change goes entirely through row-height + non-flex column scaling). A numeric N >= 1 sets a custom cap (e.g. 1.5 is conservative; Inf removes the cap).
scale numeric(1) 2 Raster fidelity multiplier for PNG output (2 = retina). PNG-only: passing scale to a .svg or .pdf save warns.
which string / integer / NULL NULL For SplitForest only. NULL (default) writes every subview into a directory; pass a key string ("Male__Young") or 1-based integer index to extract a single subview to file and demote the path to a single-file write.
paginate PaginateSpec / shortcut / NULL NULL Override the spec’s pagination at the call site. Accepts NULL (off), TRUE, a positive integer, or a PaginateSpec from paginate_spec(). When set on a .pdf save, emits a multi-page PDF; on a .png / .svg save, warns and flattens (use paginate = NULL to silence). See Pagination in the Export guide.
... — — Reserved for future format-specific options.

Returns: the output file path, invisibly. For paginated PDF output without qpdf, returns the character vector of numbered-series paths.

Sizing modes

width height ratio Mode Behavior
— — — Natural Render at the spec’s natural dimensions.
set — — Display-scaled Render at natural; SVG root width = requested. Height computed from natural aspect.
— set — Display-scaled Render at natural; SVG root height = requested. Width computed from natural aspect.
set set — Aspect-changed Relayout: target aspect = width / height. Flex columns absorb width delta (capped at flex); row heights scale for height delta.
set — set Aspect-changed Target height = width / ratio.
— set set Aspect-changed Target width = height * ratio.
— — set Aspect-changed Target width = natural; target height = natural width / ratio.
set set set Error Over-specified — pick at most two.

The flex argument controls what happens only in Aspect-changed mode: it caps how far each flex-eligible column may stretch before the layout falls back to scaling non-flex columns and row heights. Default-natural and display-scaled renders are unaffected by flex.

Dependencies

  • SVG: V8 (always required for the generator).
  • PDF / PNG: also requires the rsvg package.
  • Multi-page PDF (paginated): also requires the qpdf package, used to merge per-page PDFs into one. If qpdf is not installed, save_plot() falls back to writing a numbered series (out_p01.pdf, out_p02.pdf, …) and emits a warning.

There is no headless-browser dependency.

Resolution guide for PNG

scale is a raster-fidelity multiplier — it grows the PNG’s pixel count without changing the layout’s logical size. Vector formats (SVG / PDF) ignore scale; pass width to make a vector output denser instead.

scale Effective DPI Use case
1 ~72 Quick preview
2 ~144 Standard web / screen (default)
3 ~216 Print quality
4 ~288 High-resolution print

Examples

p <- forest_plot(data, point = "hr", lower = "lo", upper = "hi", label = "study")

save_plot(p, "figures/forest.svg")                          # natural
save_plot(p, "figures/forest.svg", width = 1200)            # display-scaled
save_plot(p, "figures/forest.svg", ratio = 2)               # 2:1 wide
save_plot(p, "figures/forest.svg", width = 1200, height = 600)  # explicit aspect
save_plot(p, "figures/forest.svg", ratio = 2, flex = FALSE)     # no flex absorption

save_plot(p, "figures/forest.png", width = 1200, scale = 3)     # 3600 px PNG

# Long table — multi-page PDF (requires qpdf in Suggests)
save_plot(p, "figures/forest.pdf", paginate = paginate_letter())

# Force a denser pagination on this render only
save_plot(p, "figures/dense.pdf",
          paginate = paginate_spec(rows = 50))

# Flatten a paginated spec for a single-image PNG without warning
save_plot(p, "figures/single.png", paginate = NULL)

# SplitForest — extract one subview to a single file
save_plot(split_p, "figures/male.svg", which = "Male")

set_aspect_ratio()

Pin an aspect ratio on the spec itself, so every render — save_plot(), the htmlwidget, the interactive aspect slider — picks it up by default. Backed by two WebSpec fields:

  • target_aspect — the numeric ratio (width / height), or NA_real_ for natural.
  • target_aspect_anchor — "width" / "height" / "auto", controls how target dims fall out of ratio.

save_plot(..., ratio = X, anchor = ...) overrides the spec-level values for one render only.

set_aspect_ratio(x, ratio, anchor = c("width", "height", "auto"))
Argument Type Description
x WebSpec / htmlwidget / tabviz_proxy The spec or live widget to modify.
ratio numeric(1) / NULL Target aspect ratio (width / height). NULL clears the target — equivalent to “render at natural aspect”.
anchor choice Resolution rule when only ratio is given. "width" (default): target width = natural width, target height = natural width / ratio. "height": anchors height instead — wide ratios grow output width rather than shrinking row height. "auto": picks based on whether the requested ratio is wider or taller than the spec’s natural aspect — readability-first; recommended. 
# 2:1 wide, with row height preserved (output grows wider)
spec |> set_aspect_ratio(2, anchor = "auto")

The in-widget aspect-ratio slider (in the toolbar’s zoom dropdown) drives the same fields live; its “Readable rows” toggle flips the anchor between "width" and "auto". See Interactivity for the full surface.

tabviz_natural_dimensions()

Compute a spec’s natural width / height / aspect without rendering — useful for sizing Quarto / RMarkdown chunks.

tabviz_natural_dimensions(x)

Returns a named list with three numeric scalars: width, height, aspect (= width / height). Accepts a WebSpec or an htmlwidget produced by tabviz() / forest_plot().

spec <- tabviz(data, label = "study", columns = list(viz_forest("hr", "lo", "hi")))
dims <- tabviz_natural_dimensions(spec)

#| fig-width: 8
#| fig-height: !expr 8 / dims$aspect

save_split_table()

Export every sub-plot in a SplitForest to a directory tree mirroring the split hierarchy. Each subview’s aspect contract is independent — width / height / ratio / flex apply per-subview.

save_split_table(x, path,
                 format = c("svg", "pdf", "png"),
                 width = NULL, height = NULL, ratio = NULL,
                 flex = TRUE, scale = 2,
                 paginate = NULL,
                 ...)
Argument Type Default Description
x SplitForest / htmlwidget required A split-forest container or a widget produced with split_by.
path character(1) required Output directory; created if it doesn’t exist.
format choice "svg" "svg", "pdf", or "png".
width / height / ratio / flex / scale — as in save_plot() Forwarded to save_plot() for each sub-plot.
paginate PaginateSpec / shortcut / NULL NULL Override the per-spec pagination at the call site. Cascades into every subview. With format = "pdf", each split leaf becomes one combined multi-page PDF (e.g. path/Male.pdf containing all of Male’s pages).
... — — Reserved for future format-specific options.

Returns: a character vector of exported file paths, invisibly.

Output directory structure

Split Files
split_by = "region" path/North.svg, path/South.svg, …
split_by = c("region", "age") path/North/North_Young.svg, path/North/North_Old.svg, …

Example

data |>
  tabviz(label = "study",
         columns = list(viz_forest(point = "or", lower = "lower", upper = "upper"))) |>
  split_table(by = c("sex", "age_group")) |>
  save_split_table("output/plots", format = "svg")

save_plot() itself accepts a SplitForest and dispatches to save_split_table() automatically — pass either a directory path or a path with an extension to choose the format.

Mutually exclusive with passing both `width` and `height`. |

### Sizing modes

| `width` | `height` | `ratio` | Mode | Behavior |
|---|---|---|---|---|
| — | — | — | **Natural** | Render at the spec's natural dimensions. |
| set | — | — | **Display-scaled** | Render at natural; SVG root width = requested. Height computed from natural aspect. |
| — | set | — | **Display-scaled** | Render at natural; SVG root height = requested. Width computed from natural aspect. |
| set | set | — | **Aspect-changed** | Relayout: target aspect = `width / height`. Flex columns absorb width delta (capped at `flex`); row heights scale for height delta. |
| set | — | set | **Aspect-changed** | Target height = `width / ratio`. |
| — | set | set | **Aspect-changed** | Target width = `height * ratio`. |
| — | — | set | **Aspect-changed** | Target width = natural; target height = natural width / ratio. |
| set | set | set | **Error** | Over-specified — pick at most two. |

## `set_aspect_ratio()` {#set_aspect_ratio}

Pin an aspect ratio on the spec itself, so every render — `save_plot()`, the htmlwidget, the interactive aspect slider — picks it up by default. Backed by two `WebSpec` fields:

* `target_aspect` — the numeric ratio (`width / height`), or `NA_real_` for natural.
* `target_aspect_anchor` — `"width"` / `"height"` / `"auto"`, controls how target dims fall out of `ratio`.

`save_plot(..., ratio = X, anchor = ...)` overrides the spec-level values for one render only.

```r
set_aspect_ratio(x, ratio, anchor = c("width", "height", "auto"))
```

| Argument | Type | Description |
|---|---|---|
| `x` | `WebSpec` / htmlwidget / `tabviz_proxy` | The spec or live widget to modify. |
| `ratio` | `numeric(1)` / `NULL` | Target aspect ratio (`width / height`). `NULL` clears the target — equivalent to "render at natural aspect". |
| `anchor` | choice | Resolution rule when only `ratio` is given. `"width"` *(default)*: target width = natural width, target height = natural width / ratio. `"height"`: anchors height instead — wide ratios grow output width rather than shrinking row height. `"auto"`: picks based on whether the requested ratio is wider or taller than the spec's natural aspect — readability-first; recommended. |

```r
# 2:1 wide, with row height preserved (output grows wider)
spec |> set_aspect_ratio(2, anchor = "auto")
```

The in-widget aspect-ratio slider (in the toolbar's zoom dropdown) drives the same fields live; its "Readable rows" toggle flips the anchor between `"width"` and `"auto"`. See [Interactivity](../guide/interactivity.qmd#aspect-ratio-slider) for the full surface.

## `save_split_table()` {#save_split_table}

Export every sub-plot in a `SplitForest` to a directory tree mirroring the split hierarchy. Each subview's aspect contract is independent — `width` / `height` / `ratio` / `flex` apply per-subview.

```r
save_split_table(x, path,
                 format = c("svg", "pdf", "png"),
                 width = NULL, height = NULL, ratio = NULL,
                 flex = TRUE, scale = 2,
                 paginate = NULL,
                 ...)
```

| Argument | Type | Default | Description |
|---|---|---|---|
| `x` | `SplitForest` / htmlwidget | required | A split-forest container or a widget produced with `split_by`. |
| `path` | `character(1)` | required | Output directory; created if it doesn't exist. |
| `format` | choice | `"svg"` | `"svg"`, `"pdf"`, or `"png"`. |
| `width` / `height` / `ratio` / `flex` / `scale` | — | as in `save_plot()` | Forwarded to `save_plot()` for each sub-plot. |
| `paginate` | `PaginateSpec` / shortcut / `NULL` | `NULL` | Override the per-spec pagination at the call site. Cascades into every subview. With `format = "pdf"`, each split leaf becomes one combined multi-page PDF (e.g. `path/Male.pdf` containing all of Male's pages). |
| `...` | — | — | Reserved for future format-specific options. |

**Returns:** a character vector of exported file paths, invisibly.

### Output directory structure

| Split | Files |
|---|---|
| `split_by = "region"` | `path/North.svg`, `path/South.svg`, … |
| `split_by = c("region", "age")` | `path/North/North_Young.svg`, `path/North/North_Old.svg`, … |

### Example

```r
data |>
  tabviz(label = "study",
         columns = list(viz_forest(point = "or", lower = "lower", upper = "upper"))) |>
  split_table(by = c("sex", "age_group")) |>
  save_split_table("output/plots", format = "svg")
```

`save_plot()` itself accepts a `SplitForest` and dispatches to `save_split_table()` automatically — pass either a directory path or a path with an extension to choose the format.
