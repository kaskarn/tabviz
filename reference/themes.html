Themes

Built-in theme presets, the web_theme() custom-theme constructor, the four core modifiers (set_inputs(), set_variants(), set_spacing(), set_theme_field()), and package_themes() for listing every shipped theme. Pass any theme to tabviz(theme = ...) or apply it after the fact with set_theme().

For the conceptual model see the Themes guide and the Styling Cascade concepts page.

Preset themes

web_theme_*() builders return a fully resolved WebTheme. They take no arguments — chain modifiers to customize.

web_theme_cochrane() The package default Cochrane teal + warm-coral accent, Inter sans-serif, comfortable density

web_theme()

Custom-theme constructor. Extends a base_theme with named lists of Tier 1 inputs and per-table variants. Resolves the theme before returning.

web_theme(
  name = "custom",
  inputs = NULL,        # Named list of ThemeInputs property overrides
  variants = NULL,      # Named list of ThemeVariants property overrides
  base_theme = web_theme_cochrane()
)
Argument Type Default Description
name string "custom" Theme name (shown in the in-widget switcher and in source export).
inputs named list NULL Tier 1 input overrides. Any ThemeInputs field is valid: neutral, brand, brand_deep, accent, accent_deep, status_*, series_anchors, font_body, font_display, font_mono. Unset fields inherit from base_theme.
variants named list NULL Per-table variant overrides: density (compact / comfortable / spacious), header_style (light / bold), first_column_style (default / bold).
base_theme WebTheme web_theme_cochrane() Starting point. The inputs and variants lists are applied on top.

Example — Lancet-inspired theme with bold headers:

custom <- web_theme(
  name = "navy-bold",
  inputs = list(
    brand          = "#1F3A5F",
    brand_deep     = "#0F2138",
    accent         = "#B08938",
    series_anchors = c("#1F3A5F", "#B08938", "#6B8E7F"),
    font_body      = "Charter, Georgia, serif"
  ),
  variants = list(density = "comfortable", header_style = "bold")
)

Theme modifiers

All modifiers operate on a WebTheme and return a new (re-resolved) WebTheme. Chainable via the pipe.

set_inputs()

Update Tier 1 customer-facing inputs. Cascade-aware: changing brand clears brand_deep so it re-mirrors; changing series_anchors rebuilds the slot bundles. The pooled-effect diamond reads from series[1], so anchoring it is just a matter of setting the first series_anchors entry.

set_inputs(
  theme,
  neutral, brand, brand_deep, accent, accent_deep,
  status_positive, status_negative, status_warning, status_info,
  series_anchors,
  font_body, font_display, font_mono
)
Argument Type Description
theme WebTheme Theme to modify
neutral chr[5] 5-step ramp, lightest → darkest
brand hex Brand fill color (markers, accents)
brand_deep hex Dark companion for text/strokes; defaults to brand
accent hex Chrome accent (hover, selection, links)
accent_deep hex Dark companion; defaults to accent
status_positive, status_negative, status_warning, status_info hex Semantic status colors
series_anchors chr[] Per-effect anchor colors (variable length). The first anchor also drives the pooled-effect diamond.
font_body string Primary CSS font-family
font_display string Optional title face; defaults to font_body
font_mono string Optional monospace face

set_variants()

Flip per-table toggles. Density change resets spacing tokens so the new preset’s values flow through.

set_variants(theme, density, header_style, first_column_style)
Argument Values Description
density "compact" / "comfortable" / "spacious" Drives every spacing token
header_style "light" / "bold" Picks between light header on muted surface and bold header on brand_deep
first_column_style "default" / "bold" Excel-style emphasized first column when "bold"

set_spacing()

Per-token spacing override on top of the active density preset. Set what you care about; the rest cascade from density.

set_spacing(
  theme,
  row_height, header_height, padding, container_padding, axis_gap,
  column_group_padding, row_group_padding, cell_padding_x,
  footer_gap, title_subtitle_gap, bottom_margin, indent_per_level
)

All values are numeric (pixels).

set_theme_field()

Generic deep setter. Walks the theme tree by path and replaces the leaf with value, then re-resolves. Use this for any field not covered by the typed setters above.

set_theme_field(theme, path, value)
Argument Description
theme A WebTheme
path Character or mixed character/integer vector. Strings are property names; integers index list-typed properties (currently only series).
value New value for the leaf field

Examples:

# Pin row-group L1 background regardless of theme
theme |> set_theme_field(c("row_group", "L1", "bg"), "#EEEEEE")

# Override series 2's stroke without touching anything else
theme |> set_theme_field(c("series", 2L, "stroke"), "#444444")

# Set Tier 1 inputs surgically
theme |> set_theme_field(c("inputs", "brand"), "#FF0000")

# Tweak axis config
theme |> set_theme_field(c("axis", "gridlines"), TRUE)
theme |> set_theme_field(c("axis", "tick_count"), 5)

# Pin layout config
theme |> set_theme_field(c("layout", "container_border_radius"), 4)

Re-resolution preserves explicit overrides on subsequent edits. set_inputs(brand = "...") after pinning surface.base will refresh series colors but leave surface.base pinned.

Switching themes on a spec

Verb Operates on Purpose
set_theme() WebSpec, htmlwidget, proxy Swap to a preset name or WebTheme object
selectable_themes() WebSpec, htmlwidget Curate the in-widget theme switcher 
# Apply by name
plot |> set_theme("jama")

# Apply by object
custom <- web_theme(inputs = list(brand = "#1F3A5F"))
plot |> set_theme(custom)

Theme structure (S7)

A WebTheme is an S7 object with the following top-level properties. For the conceptual model and how the cascades compose, see Styling Cascade and S7 Architecture.

Property Type Tier Description
name string — Theme identifier
inputs ThemeInputs 1 Customer-facing inputs
variants ThemeVariants 1 Per-table toggles
surface Surfaces 2 base / muted / raised
content Content 2 primary / secondary / muted / inverse
divider Dividers 2 subtle / strong
accent AccentRoles 2 default / muted / tint_subtle / tint_medium
status StatusColors 2 positive / negative / warning / info
series list<SlotBundle> 2 Per-anchor 7-field bundle
summary SlotBundle 2 Pooled-effect diamond bundle
text TextRoles 2 8 named role bundles
spacing SpacingTokens 2 12 density-derived numerics
annotation AnnotationCluster 3 title / subtitle / caption / footnote
header HeaderCluster 3 light + bold variants
column_group ColumnGroupCluster 3 Column-group header
row_group RowGroupCluster 3 L1 / L2 / L3 tiers
row RowCluster 3 base / alt / hover / selected + emphasis / muted / accent
cell CellCluster 3 bg / fg / border / text
first_column FirstColumnCluster 3 plain + bold variants
plot PlotScaffold 3 axis / tick / gridline / reference + mark sizes
marks MarksRecipes 3 Per-mark slot-bundle recipes
axis AxisConfig config Forest x-axis range/ticks/grid
layout Layout config Plot width, container border

## Theme modifiers

All modifiers operate on a `WebTheme` and return a new (re-resolved) `WebTheme`.
Chainable via the pipe.

### `set_inputs()` {#set_inputs}

Update Tier 1 customer-facing inputs. Cascade-aware: changing `brand` clears
`brand_deep` so it re-mirrors; changing `series_anchors` rebuilds the slot
bundles. The pooled-effect diamond reads from `series[1]`, so anchoring it
is just a matter of setting the first `series_anchors` entry.

```r
set_inputs(
  theme,
  neutral, brand, brand_deep, accent, accent_deep,
  status_positive, status_negative, status_warning, status_info,
  series_anchors,
  font_body, font_display, font_mono
)
```

| Argument | Type | Description |
|---|---|---|
| `theme` | `WebTheme` | Theme to modify |
| `neutral` | `chr[5]` | 5-step ramp, lightest → darkest |
| `brand` | hex | Brand fill color (markers, accents) |
| `brand_deep` | hex | Dark companion for text/strokes; defaults to `brand` |
| `accent` | hex | Chrome accent (hover, selection, links) |
| `accent_deep` | hex | Dark companion; defaults to `accent` |
| `status_positive`, `status_negative`, `status_warning`, `status_info` | hex | Semantic status colors |
| `series_anchors` | `chr[]` | Per-effect anchor colors (variable length). The first anchor also drives the pooled-effect diamond. |
| `font_body` | string | Primary CSS font-family |
| `font_display` | string | Optional title face; defaults to `font_body` |
| `font_mono` | string | Optional monospace face |

### `set_variants()` {#set_variants}

Flip per-table toggles. Density change resets spacing tokens so the new
preset's values flow through.

```r
set_variants(theme, density, header_style, first_column_style)
```

| Argument | Values | Description |
|---|---|---|
| `density` | `"compact"` / `"comfortable"` / `"spacious"` | Drives every spacing token |
| `header_style` | `"light"` / `"bold"` | Picks between light header on muted surface and bold header on `brand_deep` |
| `first_column_style` | `"default"` / `"bold"` | Excel-style emphasized first column when `"bold"` |

### `set_spacing()` {#set_spacing}

Per-token spacing override on top of the active density preset. Set what you

All values are numeric (pixels).

### `set_theme_field()` {#set_theme_field}

Generic deep setter. Walks the theme tree by `path` and replaces the leaf
with `value`, then re-resolves. Use this for any field not covered by the
typed setters above.

```r
set_theme_field(theme, path, value)
```

| Argument | Description |
|---|---|
| `theme` | A `WebTheme` |
| `path` | Character or mixed character/integer vector. Strings are property names; integers index list-typed properties (currently only `series`). |
| `value` | New value for the leaf field |

Examples:

## Theme structure (S7)

A `WebTheme` is an S7 object with the following top-level properties.
For the conceptual model and how the cascades compose, see
[Styling Cascade](../concepts/styling-cascade.qmd) and
[S7 Architecture](../technical/s7-classes.qmd).

| Property | Type | Tier | Description |
|---|---|---|---|
| `name` | string | — | Theme identifier |
| `inputs` | `ThemeInputs` | 1 | Customer-facing inputs |
| `variants` | `ThemeVariants` | 1 | Per-table toggles |
| `surface` | `Surfaces` | 2 | base / muted / raised |
| `content` | `Content` | 2 | primary / secondary / muted / inverse |
| `divider` | `Dividers` | 2 | subtle / strong |
| `accent` | `AccentRoles` | 2 | default / muted / tint_subtle / tint_medium |
| `status` | `StatusColors` | 2 | positive / negative / warning / info |
| `series` | `list<SlotBundle>` | 2 | Per-anchor 7-field bundle |
| `summary` | `SlotBundle` | 2 | Pooled-effect diamond bundle |
| `text` | `TextRoles` | 2 | 8 named role bundles |
| `spacing` | `SpacingTokens` | 2 | 12 density-derived numerics |
| `annotation` | `AnnotationCluster` | 3 | title / subtitle / caption / footnote |
| `header` | `HeaderCluster` | 3 | light + bold variants |
| `column_group` | `ColumnGroupCluster` | 3 | Column-group header |
| `row_group` | `RowGroupCluster` | 3 | L1 / L2 / L3 tiers |
| `row` | `RowCluster` | 3 | base / alt / hover / selected + emphasis / muted / accent |
| `cell` | `CellCluster` | 3 | bg / fg / border / text |
| `first_column` | `FirstColumnCluster` | 3 | plain + bold variants |
| `plot` | `PlotScaffold` | 3 | axis / tick / gridline / reference + mark sizes |
| `marks` | `MarksRecipes` | 3 | Per-mark slot-bundle recipes |
| `axis` | `AxisConfig` | config | Forest x-axis range/ticks/grid |
| `layout` | `Layout` | config | Plot width, container border |
