Reference for every column-defining helper: data columns (col_*()), per-row visualization columns (viz_*()), the low-level web_col() constructor, column groups, and the per-effect helpers used by multi-effect viz columns.

Importantfield is now a required argument

col_n(), col_bar(), col_pvalue(), and col_sparkline() previously had implicit defaults for field ("n", "weight", "pvalue", "trend"). Those defaults are soft-deprecated since 0.9.0 — pass the column name explicitly. Other col_*() helpers have always required field.

All col_*() helpers also accept the cell-styling arguments documented in Cell-level styling below: bold, italic, color, bg, badge, icon, emphasis, muted, accent, plus align, header_align, wrap, sortable, na_text, tooltip, and formatter.

Notetooltip and formatter (added in 0.9.0)
  • tooltip = "<col>" — column whose values are the per-cell hover text
  • formatter = function(x) ... — escape hatch for custom value rendering. The function receives the raw column vector and must return a character vector of the same length; the output is rendered as text. Use this when the built-in numeric/interval/percent formatting can’t express the desired output.

Basic data columns

col_text()

Plain text column.

col_text(field, header = NULL, width = NULL, max_chars = NULL, ...)
Argument Type Default Description
field character(1) required Data column name.
header character(1) NULL (uses field) Display header.
width numeric(1) NULL (auto) Column width in pixels.
max_chars integer(1) NULL Truncate with ellipsis past this length.
... — — Forwarded to web_col() (cell styling, alignment).

col_label()

Same as col_text() but auto-prettifies the default header (e.g. "study_id" → "Study Id"). Useful for the leftmost row-identifier column when you want explicit control.

col_label(field, header = NULL, width = NULL, max_chars = NULL, ...)

col_numeric()

Numeric column with formatting, sig-fig support, large-number abbreviation.

col_interval()

             imprecise_threshold = NULL,
             na_text = NULL, ...)
Argument Type Default Description
point character(1) required Point estimate column.

upper character(1) required Upper bound column.

digits integer(1) NULL Significant figures. Mutually exclusive with decimals.
thousands_sep character(1) or FALSE FALSE Thousands separator string or FALSE.
abbreviate logical(1) FALSE Shorten large values ("1.2K").

  • sep was renamed to separator; the old name still works with a deprecation warning.

Per-row visualization columns (viz_*)

These columns plot one image per row with a shared axis across rows. They define their own effects and are not sortable.

Low-level constructor

web_col()

The constructor every col_*() helper delegates to. Use this directly only when you need a column type the helpers don’t cover.

web_col(field,
        header = NULL,
        type = c("text", "numeric", "interval", "bar", "pvalue",
                 "sparkline", "icon", "badge", "stars", "img",
                 "reference", "range", "forest", "heatmap",
                 "progress", "viz_bar", "viz_boxplot",
                 "viz_violin", "custom"),
        width = NULL,
        align = NULL, header_align = NULL,
        show_header = NULL,
        wrap = FALSE, sortable = TRUE,
        options = list(),
        na_text = NULL,
        # Cell styling
        bold = NULL, italic = NULL,
        color = NULL, bg = NULL,
        badge = NULL, icon = NULL,
        emphasis = NULL, muted = NULL, accent = NULL)

Returns a ColumnSpec.

Cell-level styling

Every col_*() helper accepts the same set of cell-styling arguments via .... Each takes either a column name (resolved at render time) or a formula evaluated against the row (use .x to refer to the column’s own value).

Argument Resolved value Effect
bold logical Bold text.
italic logical Italic text.
color CSS color Text color.
bg CSS color Cell background.
badge character Inline badge in the cell.
icon emoji/unicode Icon next to value.
emphasis logical Theme-aware emphasis (bold + foreground).
muted logical Theme-aware muted (lighter).
accent logical Theme accent color. 
# Bold significant p-values, color them
col_pvalue("pval",
           bold  = ~ .x < 0.05,
           color = ~ ifelse(.x < 0.05, "#16a34a", "#71717a"))

# Reference another column
col_numeric("hr", "HR",
            color = ~ ifelse(pval < 0.05, "#16a34a", "#71717a"))

Alignment

Two slots: align for body cells, header_align for the header. Defaults are type-aware:

Helper Body align default
col_text, col_numeric, col_pvalue, col_bar, col_interval, col_sparkline, col_percent, col_reference, col_heatmap, col_progress, col_date "left"
col_icon, col_badge, col_stars, col_img "center"
col_range, col_currency "right"
viz_forest header "center"
viz_bar header "left"
viz_boxplot, viz_violin header "center"

header_align defaults to “follow the body” — set it explicitly to decouple.

Examples

Code 
data <- data.frame(
  study  = c("Trial A", "Trial B", "Trial C"),
  hr     = c(0.72, 0.85, 0.68),
  lower  = c(0.55, 0.70, 0.52),
  upper  = c(0.95, 1.03, 0.89),
  n      = c(245, 180, 320),
  weight = c(35, 28, 37),
  pval   = c(0.018, 0.092, 0.003)
)

tabviz(data,
  label = "study",
  columns = list(
    col_n("n"),
    col_bar("weight"),
    viz_forest(point = "hr", lower = "lower", upper = "upper",
               scale = "log", null_value = 1),
    col_interval("hr", "lower", "upper", header = "HR (95% CI)"),
    col_pvalue("pval", stars = TRUE)
  )
)
# Column groups
columns = list(
  col_n("n"),
  col_group("Treatment Effect",
    col_numeric("hr", "HR"),
    col_interval("hr", "lower", "upper", header = "95% CI")
  ),
  col_pvalue("pval")
)

## Basic data columns

### `col_text()` {#col_text}

Plain text column.

```r
col_text(field, header = NULL, width = NULL, max_chars = NULL, ...)
```

| Argument | Type | Default | Description |
|---|---|---|---|
| `field` | `character(1)` | required | Data column name. |
| `header` | `character(1)` | `NULL` (uses `field`) | Display header. |
| `width` | `numeric(1)` | `NULL` (auto) | Column width in pixels. |
| `max_chars` | `integer(1)` | `NULL` | Truncate with ellipsis past this length. |
| `...` | — | — | Forwarded to `web_col()` (cell styling, alignment). |

### `col_label()` {#col_label}

Same as `col_text()` but auto-prettifies the default header (e.g. `"study_id"` → `"Study Id"`). Useful for the leftmost row-identifier column when you want explicit control.

```r
col_label(field, header = NULL, width = NULL, max_chars = NULL, ...)
```

### `col_numeric()` {#col_numeric}

Numeric column with formatting, sig-fig support, large-number abbreviation.

```r
col_numeric(field, header = NULL, width = NULL,
            decimals = 2, digits = NULL,
            thousands_sep = FALSE, abbreviate = FALSE, ...)
```

| Argument | Type | Default | Description |
|---|---|---|---|
| `field` | `character(1)` | required | Data column. |
| `header` | `character(1)` | `NULL` | Header text. |
| `width` | `numeric(1)` | `NULL` | Width in px. |
| `decimals` | `integer(1)` | `2` | Fixed decimal places. Mutually exclusive with `digits`. |
| `digits` | `integer(1)` | `NULL` | Significant figures. |

### `col_pvalue()` {#col_pvalue}


| Argument | Type | Default | Description |
|---|---|---|---|

## Cell-level styling {#cell-level-styling}

Every `col_*()` helper accepts the same set of cell-styling arguments via `...`. Each takes either a **column name** (resolved at render time) or a **formula** evaluated against the row (use `.x` to refer to the column's own value).

| Argument | Resolved value | Effect |
|---|---|---|
| `bold` | logical | Bold text. |
| `italic` | logical | Italic text. |
| `color` | CSS color | Text color. |
| `bg` | CSS color | Cell background. |
| `badge` | character | Inline badge in the cell. |
| `icon` | emoji/unicode | Icon next to value. |
| `emphasis` | logical | Theme-aware emphasis (bold + foreground). |
| `muted` | logical | Theme-aware muted (lighter). |
| `accent` | logical | Theme accent color. |

```r
# Bold significant p-values, color them
col_pvalue("pval",
           bold  = ~ .x < 0.05,
           color = ~ ifelse(.x < 0.05, "#16a34a", "#71717a"))

# Reference another column
col_numeric("hr", "HR",
            color = ~ ifelse(pval < 0.05, "#16a34a", "#71717a"))
```

## Alignment

Two slots: `align` for body cells, `header_align` for the header. Defaults are type-aware:

| Helper | Body `align` default |
|---|---|
| `col_text`, `col_numeric`, `col_pvalue`, `col_bar`, `col_interval`, `col_sparkline`, `col_percent`, `col_reference`, `col_heatmap`, `col_progress`, `col_date` | `"left"` |
| `col_icon`, `col_badge`, `col_stars`, `col_img` | `"center"` |
| `col_range`, `col_currency` | `"right"` |
| `viz_forest` | header `"center"` |
| `viz_bar` | header `"left"` |
| `viz_boxplot`, `viz_violin` | header `"center"` |

`header_align` defaults to "follow the body" — set it explicitly to decouple.

## Examples

```{r}
data <- data.frame(
  study  = c("Trial A", "Trial B", "Trial C"),
  hr     = c(0.72, 0.85, 0.68),
  lower  = c(0.55, 0.70, 0.52),
  upper  = c(0.95, 1.03, 0.89),
  n      = c(245, 180, 320),
  weight = c(35, 28, 37),
  pval   = c(0.018, 0.092, 0.003)
)

tabviz(data,
  label = "study",
  columns = list(
    col_n("n"),
    col_bar("weight"),
    viz_forest(point = "hr", lower = "lower", upper = "upper",
               scale = "log", null_value = 1),
    col_interval("hr", "lower", "upper", header = "HR (95% CI)"),
    col_pvalue("pval", stars = TRUE)
  )
)
```

```r
# Column groups
columns = list(
  col_n("n"),

  ),

)
```
