Columns

  • Show All Code
  • Hide All Code
  • View Source

Reference for every column-defining helper: data columns (col_*()), per-row visualization columns (viz_*()), the low-level web_col() constructor, column groups, and the per-effect helpers used by multi-effect viz columns.

All col_*() helpers require field (the data column name) as the first positional argument. They also accept the cell-styling arguments documented in Cell-level styling below: bold, italic, color, bg, badge, icon, emphasis, muted, accent, fill, plus align, header_align, wrap, sortable, na_text, tooltip, formatter, and flex.

Noteflex — opt-in to aspect-ratio absorption

Each column carries a flex flag that tells save_plot()’s lever ladder whether the column may grow / shrink to absorb a width delta when an aspect target is pinned. Defaults:

  • viz_forest(), viz_bar(), viz_boxplot(), viz_violin() → flex = TRUE
  • everything else → flex = FALSE

Pass flex = TRUE on a non-viz column to opt in (it will scale with the forest plot under wide aspects), or flex = FALSE on a viz column to pin its width and force the height ladder to absorb the delta. The save-time save_plot(flex = N) arg caps how far each flex column may stretch — see save_plot().

Notetooltip and formatter cell args
  • tooltip = "<col>" — column whose values are the per-cell hover text
  • formatter = function(x) ... — escape hatch for custom value rendering. The function receives the raw column vector and must return a character vector of the same length; the output is rendered as text. Use this when the built-in numeric/interval/percent formatting can’t express the desired output.

col_ring()

col_ring(field, header = NULL, width = NULL,
         min_value = 0, max_value = 1,
         color = NULL, thresholds = NULL,

         show_label = TRUE,
         label_format = c("percent", "decimal", "integer"),

Donut gauge that fills (value - min_value) / (max_value - min_value) of a circle. thresholds is a numeric vector of breakpoints (raw input units) where the arc color shifts; pass a paired color vector of length length(thresholds) + 1, or omit color for the auto-defaulted positive/warning/negative palette (with up to two thresholds).

Per-row visualization columns (viz_*)

These columns plot one image per row with a shared axis across rows. They define their own effects and are not sortable.

Low-level constructor

web_col()

The constructor every col_*() helper delegates to. Use this directly only when you need a column type the helpers don’t cover.

web_col(field,
        header = NULL,
        type = c("text", "numeric", "interval", "bar", "pvalue",
                 "sparkline", "icon", "badge", "stars", "pictogram",
                 "ring", "img", "reference", "range", "forest",
                 "heatmap", "progress", "viz_bar", "viz_boxplot",
                 "viz_violin", "custom"),
        width = NULL,
        align = NULL, header_align = NULL,
        show_header = NULL,
        wrap = FALSE, sortable = TRUE,
        options = list(),
        na_text = NULL,
        # Cell styling
        bold = NULL, italic = NULL,
        color = NULL, bg = NULL,
        badge = NULL, icon = NULL,
        emphasis = NULL, muted = NULL, accent = NULL,
        fill = NULL)

Returns a ColumnSpec.

Cell-level styling

Every col_*() helper accepts the same set of cell-styling arguments via .... Each takes either a column name (resolved at render time) or a formula evaluated against the row (use .x to refer to the column’s own value).

Argument Resolved value Effect
bold logical Bold text (literal weight).
italic logical Italic text.
color CSS color Text color (literal).
bg CSS color Cell background (literal).
badge character Inline badge in the cell.
icon emoji/unicode Icon next to value.
emphasis (semantic) logical Bold + theme primary color.
muted (semantic) logical Lighter color, reduced prominence.
accent (semantic) logical Bold + theme accent color.
fill (semantic) logical Bold + strong row-fill background; fg auto-contrasts. 
# Bold significant p-values, color them
col_pvalue("pval",
           bold  = ~ .x < 0.05,
           color = ~ ifelse(.x < 0.05, "#16a34a", "#71717a"))

# Reference another column
col_numeric("hr", "HR",
            color = ~ ifelse(pval < 0.05, "#16a34a", "#71717a"))

Alignment

Two slots: align for body cells, header_align for the header. Defaults are type-aware:

Helper Body align default
col_text, col_numeric, col_pvalue, col_bar, col_interval, col_sparkline, col_percent, col_reference, col_heatmap, col_progress, col_date "left"
col_icon, col_badge, col_stars, col_img "center"
col_range, col_currency "right"
viz_forest header "center"
viz_bar header "left"
viz_boxplot, viz_violin header "center"

header_align defaults to “follow the body” — set it explicitly to decouple.

Examples

Code 
data <- data.frame(
  study  = c("Trial A", "Trial B", "Trial C"),
  hr     = c(0.72, 0.85, 0.68),
  lower  = c(0.55, 0.70, 0.52),
  upper  = c(0.95, 1.03, 0.89),
  n      = c(245, 180, 320),
  weight = c(35, 28, 37),
  pval   = c(0.018, 0.092, 0.003)
)

tabviz(data,
  label = "study",
  columns = list(
    col_n("n"),
    col_bar("weight"),
    viz_forest(point = "hr", lower = "lower", upper = "upper",
               scale = "log", null_value = 1),
    col_interval("hr", "lower", "upper", header = "HR (95% CI)"),
    col_pvalue("pval", stars = TRUE)
  )
)
# Column groups
columns = list(
  col_n("n"),
  col_group("Treatment Effect",
    col_numeric("hr", "HR"),
    col_interval("hr", "lower", "upper", header = "95% CI")
  ),
  col_pvalue("pval")
)

## Cell-level styling {#cell-level-styling}

Every `col_*()` helper accepts the same set of cell-styling arguments via `...`. Each takes either a **column name** (resolved at render time) or a **formula** evaluated against the row (use `.x` to refer to the column's own value).

| Argument | Resolved value | Effect |
|---|---|---|
| `bold` | logical | Bold text (literal weight). |
| `italic` | logical | Italic text. |
| `color` | CSS color | Text color (literal). |
| `bg` | CSS color | Cell background (literal). |
| `badge` | character | Inline badge in the cell. |
| `icon` | emoji/unicode | Icon next to value. |
| `emphasis` *(semantic)* | logical | Bold + theme primary color. |
| `muted` *(semantic)* | logical | Lighter color, reduced prominence. |
| `accent` *(semantic)* | logical | Bold + theme accent color. |
| `fill` *(semantic)* | logical | Bold + strong row-fill background; `fg` auto-contrasts. |

```r
# Bold significant p-values, color them
col_pvalue("pval",
           bold  = ~ .x < 0.05,
           color = ~ ifelse(.x < 0.05, "#16a34a", "#71717a"))

# Reference another column
col_numeric("hr", "HR",
            color = ~ ifelse(pval < 0.05, "#16a34a", "#71717a"))
```

## Alignment

Two slots: `align` for body cells, `header_align` for the header. Defaults are type-aware:

| Helper | Body `align` default |
|---|---|
| `col_text`, `col_numeric`, `col_pvalue`, `col_bar`, `col_interval`, `col_sparkline`, `col_percent`, `col_reference`, `col_heatmap`, `col_progress`, `col_date` | `"left"` |
| `col_icon`, `col_badge`, `col_stars`, `col_img` | `"center"` |
| `col_range`, `col_currency` | `"right"` |
| `viz_forest` | header `"center"` |
| `viz_bar` | header `"left"` |
| `viz_boxplot`, `viz_violin` | header `"center"` |

`header_align` defaults to "follow the body" — set it explicitly to decouple.

## Examples

```{r}
data <- data.frame(
  study  = c("Trial A", "Trial B", "Trial C"),
  hr     = c(0.72, 0.85, 0.68),
  lower  = c(0.55, 0.70, 0.52),
  upper  = c(0.95, 1.03, 0.89),
  n      = c(245, 180, 320),
  weight = c(35, 28, 37),
  pval   = c(0.018, 0.092, 0.003)
)

tabviz(data,
  label = "study",
  columns = list(
    col_n("n"),
    col_bar("weight"),
    viz_forest(point = "hr", lower = "lower", upper = "upper",
               scale = "log", null_value = 1),
    col_interval("hr", "lower", "upper", header = "HR (95% CI)"),
    col_pvalue("pval", stars = TRUE)
  )
)
```

```r
# Column groups
columns = list(
  col_n("n"),
  col_group("Treatment Effect",
    col_numeric("hr", "HR"),
    col_interval("hr", "lower", "upper", header = "95% CI")
  ),
  col_pvalue("pval")
)
```
